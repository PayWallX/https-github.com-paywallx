# Demo Overview

This document explains what the PayWallX demo showcases and its limitations.

## What the demo shows

- A protected resource returning `HTTP 402 â€” Payment Required`
- A clear paywall UI with a one-time price
- A simulated or testnet crypto payment (SOL / USDC)
- Instant access after payment
- Transition from HTTP 402 to HTTP 200

## What the demo does NOT show

- Production-ready APIs
- Mainnet payment verification
- Subscription management
- User accounts or authentication

## Purpose

The demo exists to validate the core concept:

**Payment is the access control layer.**
